# MDT 安全风控系统

对应脚本：`mdtserver/config/scripts/wayzer/security/securityGuard.kts`

## 目标

用于处理机器人/刷屏/刷菜单/短时间多连接等异常行为，和现有账号/IP防熊、上行监控、性能优化系统配合使用。

## 当前功能

- 单 IP 连接频率与短时间多 UUID/USID 检测。
- 聊天/指令限速。
  - 未登录玩家：仍按 IP 使用较低阈值，默认 `5/5s`。
  - 已登录玩家：按玩家 UUID 使用更高阈值，默认 `20/5s`，避免同 IP 正常玩家互相误伤。
- 菜单打开限速，基于 `coreMindustry/menu.lib.kt` 的 `IMenuOpenGuard` 拦截所有菜单。
  - 未绑定玩家：仍按 IP 使用较低阈值，默认 `8/5s`，用于挡机器人快速刷菜单。
  - 已绑定玩家：切换为按玩家 UUID 计数，默认 `40/10s`，避免同 IP 正常玩家互相误伤。
  - `/help`/指令类菜单：已绑定玩家额外放宽倍率，默认 `x3`，快速翻页通常不会触发；只有持续反复刷才会拦截。
  - 原生 admin/4+admin 打开菜单不参与菜单限速处罚。
  - 菜单打开过快只走“提示后踢出”的轻处罚；不再增加异常分，也不会因此封禁账号或 IP。
- 非菜单类异常处罚阶梯：首次提示，第二次踢出，继续触发则封禁 IP 1 天。
- IP 封禁落盘到 `MdtSettings`，重启后仍生效。
- 异常分默认阈值：
  - 普通风控：`15` 分。
  - 增强风控：`30` 分。
- 自动风控持续时间默认随机 `20-60` 分钟；管理员手动设置模式时，不填分钟则使用对应模式最短时间。
- 普通风控模式：只限制风控开启后新进入、且不能自动登录的游客。
  - 新进入游客会被临时分配为观察者，并丢弃普通聊天/大部分指令。
  - 只保留登录、注册、账号、帮助、Wiki、状态、风控菜单等必要入口。
  - 已经在线的游客不会因为普通风控开启而被批量切到观察者。
  - 如确实需要恢复旧行为，可通过脚本配置 `guardForceGuestSpectate=true` 强制所有游客观战。
- 增强风控模式：只允许已登录且能自动登录的玩家进服；未登录玩家无法进服，且已在服的未登录玩家会被强制观战。
- 结合上行估算：默认不再由上行超预算自动进入安全风控，避免和 `serverPressure`/性能优化系统重复触发；如确实需要，可通过脚本配置 `trafficAutoSecurityEnabled=true` 开启。
- 风控/增强风控会在随机时间后自动解除，不广播，只写日志，并私聊通知在线4级/admin。
- `/help` 管理指令分区已加入“安全风控”。

## 指令

权限：`wayzer.admin.security`，默认 `@admin`，4级玩家因信任等级系统会获得 admin 权限组。

- `/security` 或 `/security menu`：打开管理页面，集中查看/切换模式、重置、释放观战残留、管理今日游客观战；该入口已放入 `/help` → “管理指令”。
- `/security status`：查看模式、异常分、IP封禁数、限速阈值、上行状态。
- `/security spectators`：查看当前观察者队伍玩家的来源速查；可识别安全风控、今日游客观战、风险IP，其他来源需结合 `/forceOB` 或玩家主动 `/ob` 排查。
- `/security mode normal|guard|enhanced [分钟] [原因]`：手动设置或解除风控模式。
- `/security banip <ip> [分钟] [原因]`：手动封禁 IP。
- `/banip <在线玩家3位ID/#游戏ID/名字> [分钟] [原因]`：根据在线玩家当前 IP 封禁；离线玩家可从 `/recentplayers` 最近玩家面板按最近 IP 封禁。3++协管只能处理低于3++的目标，默认单次最长7天。
- `/banips`：打开当前 IP 封禁列表菜单，显示 IP、对应 UUID、玩家名、剩余时间和原因，可点开详情解除封禁；控制台执行时输出文字列表。
- `/banlist`：统一显示玩家/账号封禁与 IP 封禁，按到期时间排序；详情页可快速调用对应的账号解封或 IP 解封。
- `/unbanip <ip>`：解除 IP 封禁；3++只能解除自己施加的记录，4级/admin可解除全部。
- `/security unbanip <ip>`：解除 IP 封禁。
- `/security check <ip/三位ID>`：查询 IP/玩家当前风控状态。
- `/security release`：只释放安全风控造成的临时游客观战残留；若今日游客观战、风险IP、投票强制观战仍生效，会继续保持观战。
- `/security reset`：清空临时计数并退出风控模式。

菜单备注：

- `/security` 页面已按功能分行：状态查看、普通/增强风控、重置/释放、观战来源/IP、今日游客观战、封禁IP分别分组，避免按钮全部挤在一行。

## 本轮误触发修正（2026-05-24）

- 日志中出现“游客被观战且聊天像被禁言，但 `/guestob status` 未开启”的情况，实际可能来自 `securityGuard` 的普通风控模式，而不是“今日游客强制观战”脚本。
- 普通风控模式现在默认只限制风控开启后新进入的游客，不再直接把已在线游客全员切到观察者；增强风控仍保留强制观战。
- 安全风控内部会记录自己临时切到观察者的游客 UUID；风控解除、自动过期、`/security reset` 或 `/security release` 会尝试释放这些观战残留。
- 管理菜单新增“查看观战来源速查”，用于排查游客/玩家为什么在观察者队伍，避免把安全风控、今日游客观战、风险IP、投票强制观战和主动观战混在一起。
- 玩家完成登录后，如果仍处于安全风控造成的临时观战，会在聊天/菜单/等级变化回调中尝试自动解除。
- 已登录玩家聊天/指令限速从“同 IP 低阈值”改为“玩家 UUID 高阈值”，降低管理连续执行 `/help`、`/playerunmute` 等指令时把同 IP 其他人一起带进风控的概率。
- 短时间连接检测改为：同 IP 主要看不同 UUID/USID 数；同一个游客反复掉线重连不会单独因为连接次数超过阈值就判定异常。
- 风控模式变化会私聊通知在线4级/admin；管理上线时如果仍处于风控，也会收到当前状态提示。
- 启动时如果旧状态是“上行触发的风控”且当前 `trafficAutoSecurityEnabled=false`，会自动清理为 normal。

别名：`/风控`、`/安全风控`。

## 与其他脚本的关系

- `wayzer/user/accountIpGuard.kts`：处理管理禁封/封禁账号进入、同 IP 投票踢出累计触发后的风险IP未登录限制，并负责同IP换号提示；提示会在玩家入服后延迟检查，等待自动登录状态稳定，广播“当前玩家可能为上次同IP玩家的小号”，同时显示上次名字、UUID 与短ID，并跳过同一游戏UUID的正常登录态切换；同一组合短时间内会去重，连接阶段只做轻量记录不额外查自动登录库。安全风控处理实时刷屏、短时间多连接和 IP 封禁，刷菜单仅做提示/踢出限速。
- `wayzer/ext/playerMute.kts` 与 `wayzer/ext/playerBuildBan.kts`：提供人工处罚类的禁言/禁建。玩家信息面板执行时可填写分钟数作为临时限制，留空表示永久；临时禁言/临时禁建仅内存保存，重启后失效。
- `wayzer/reGrief/trafficMonitor.kts`：提供估算上行数据，安全风控只读取其预算与平均上行。
- `wayzer/map/serverPressure*.kts`：负责 TPS/上行压力下的游戏性能措施；安全风控只负责玩家行为层面的限制。
- `wayzer/vote.lib.kt`：已改为同 IP 投票只计一票，并在投票失败后共享同 IP 发起冷却；游客失败冷却更长。
- `wayzer/security/forceObAutoCleaner.kts`：在线人数达到或超过阈值时，除清理投票/管理强制观战玩家外，也会清理已登录同IP多账号，只保留一个账号以释放玩家位。

## 注意事项

- 脚本级风控不能真正抵御网络层 DDoS；云防火墙、高防、系统防火墙仍是第一层。
- 同 IP 限制可能误伤宿舍/校园网/网吧用户，阈值不要设得过低。
- 菜单限速已对已绑定玩家做上限切换，不再让已绑定玩家共享游客的低 IP 阈值；如果仍出现误伤，可优先调高 `authedMenuLimit`、`authedMenuWindowMillis` 或 `helpMenuLimitMultiplier`。菜单限速不会增加异常分或触发封禁。
- 增强风控期间，新玩家无法注册/首次登录；管理可用 `/security mode normal` 手动解除。
- “游客观战”可能来自多个脚本：`securityGuard`、`accountGuestControl`、`accountIpGuard`、`voteOb` 或玩家主动观战。排查时优先使用 `/security status`、`/guestob status`、`/ipguard check <ip>`、玩家信息面板/`/forceOB` 状态分别确认来源。
