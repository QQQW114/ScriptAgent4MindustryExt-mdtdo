# 帖子系统

## 脚本路径

```text
mdtserver/config/scripts/wayzer/user/forumPosts.kts
```

## 入口

- `/posts`：打开帖子分区列表。
- `/post`、`/forum`、`/帖子`、`/帖子列表`：`/posts` 的别名。
- `/posts <帖子ID>`：直接打开指定帖子。
- `/posts <分区code>`：直接打开指定分区。
- `/posts new`：发布新帖子。
- `/posts share <帖子ID>`：把指定帖子分享到聊天栏，并提示其他玩家可用 `/posts <帖子ID>` 快速打开；等级分区帖子只会发送给有权限查看该分区的玩家，避免标题泄漏。
- `/posts history`：查看最近 10 条玩家修改/删除帖子记录。
- `/posts admin`：3+级/管理员打开分区管理。
- `/posts lock <帖子ID>`：3+级/管理员锁定帖子，防止自动清理删除。
- `/posts unlock <帖子ID>`：3+级/管理员解除帖子自动清理锁定。
- `/posts protect <帖子ID>`：4级/admin 设置帖子保护锁，4级以下不可编辑/删除。
- `/posts unprotect <帖子ID>`：4级/admin 解除帖子保护锁。
- `/posts trash`：4级/admin 打开帖子回收站。
- `/posts restore <帖子ID>`：4级/admin 从回收站恢复帖子。
- `/posts purge <帖子ID>`：4级/admin 彻底删除回收站内帖子和评论。
- `/help` → 玩家指令 → **帖子列表**：快速进入帖子系统。
- `/help` → 管理指令：已加入 **帖子管理 / 帖子回收站 / 帖子保护锁** 快捷入口；4级/admin进入帖子详情页后，会在评论按钮下方看到独立的 **设置保护锁（4）/解除保护锁（4）** 按钮。

## 定位

- Wiki：用于服务器规则、准则、长期重要内容。
- 帖子：用于玩家间交流、临时信息、灵活讨论。

帖子系统不做独立点赞体系。帖子详情页的按钮：

- `为作者点赞`
- `为作者点踩`

会直接调用现有玩家赞踩系统，受到现有每日总量、对单人每日次数、0级限制、自赞限制等规则约束，并触发现有被赞消息、MDC和晋升检测。

## 当前功能

- 帖子分区分页显示。
- 分区首页显示：
  - `当前帖子总数`：当前仍存在的正常帖子数。
  - `总发帖数`：历史累计发帖数，会随新帖子增加，不会因自动清理/删除而减少。
- 分区内帖子列表分页显示。
- 帖子正文分页显示。
- 帖子正文/评论支持 MDT 轻量格式：`#`/`##` 标题、`-`/`*` 列表、`>` 引用、`**重点**`、反引号代码、`---` 分割线、`[文字](链接)`；图片不会内嵌显示，只会转成图片链接提示。
- 输入框中可用 `\n`、`|` 或全角 `｜` 快速换行。
- 帖子分区页提供“最近变更”，显示最近 10 条玩家修改/删除帖子记录。
- 帖子详情页提供“分享到聊天”，会在聊天栏发送帖子标题、分区和 `/posts <帖子ID>` 快捷打开指令；等级分区帖子只会发送给可查看该分区的玩家。
- 1级及以上玩家可发帖、评论。
- 每名普通玩家每天最多发布 3 个帖子。
- 玩家在当前分区内发帖；如果在“全部分区”发帖，该帖子只属于 `all` 分区，不会进入其他具体分区。
- 每个帖子支持：
  - 为作者点赞
  - 为作者点踩
  - 发布评论
  - 查看评论（x条）
- 帖子作者可以修改自己帖子的标题和正文。
- 3+级玩家与管理员可以：
  - 修改任意帖子
  - 置顶/取消置顶帖子
  - 锁定/解除锁定帖子，锁定帖不会被自动清理系统删除
  - 删除帖子到回收站
  - 新增/修改帖子分区名称和介绍
- 4级/admin 可以：
  - 设置/解除帖子保护锁；受保护帖子只有4级/admin可编辑或删除
  - 查看回收站、恢复帖子、彻底删除回收站帖子
- 非4级/admin 删除帖子时需要填写理由，并受到删除频率限制，避免3+级玩家短时间批量删除帖子。

## 性能/懒加载

- 分区列表只分页显示当前页。
- 帖子列表使用数据库分页读取当前页，不再打开分区时全量加载所有帖子。
- 评论列表使用数据库分页读取当前页，不再一次性加载某帖全部评论。
- 帖子列表只对当前页帖子构建按钮文本；锁定帖 ID 会批量读取一次，避免每个帖子按钮重复查库。
- 自动清理和列表/评论数据库读取放入 `Dispatchers.IO`，减少对 Mindustry 游戏主线程的影响。

## 默认分区

当前默认分区：

- `all` / **全部分区**：显示全部的帖子。
- `mdt` / **MDT分区**：mindustry游戏有关的分区。
- `other` / **其他分区**：其他未分类内容。
- `chat` / **搞七捻三**：闲聊吹水的板块。不得讨论政治、色情等违规内容。
- `feedback` / **运营反馈**：此处可为服务器提供建议，指出bug等。
- `level1` / **1级分区**：1级及以上玩家可查看/发帖。
- `level2` / **2级分区**：2级及以上玩家可查看/发帖。
- `level3` / **3级分区**：3级及以上玩家可查看/发帖。
- `level3plus` / **3+级分区**：3+级及以上玩家可查看/发帖。

说明：

- “全部分区”是聚合入口，会显示所有分区的帖子。
- “全部分区”也是可发帖分区；在此发布的帖子分区为 `all`，不会显示到 `mdt`、`chat` 等具体分区里。
- 对等级不足的玩家，“全部分区”会自动隐藏高等级分区帖子；直接打开高等级分区或帖子时会提示等级不足，无法查看。

## 清理机制

为保护数据库空间，帖子系统会自动清理旧帖：

- 服务端启动时检查一次。
- 玩家打开帖子列表时，如果当天还未清理过，会清理一次。
- 默认最多保留 500 个普通非置顶帖子。
- 置顶帖不会被自动清理。
- 被 3+级玩家/管理员锁定的帖子不会被自动清理。
- 如果锁定帖被手动删除，锁定记录会同步移除。
- 自动清理与普通删除都会先把帖子移入回收站，不会立即硬删除；4级/admin可在回收站恢复或彻底删除。
- 只有超过上限时，才会从 30 天前的普通帖中按低互动优先清理。

清理参考项：

- 从帖子入口产生的“为作者点赞/点踩”成功次数。
- 评论数。
- 创建时间。

备注：这些互动计数只用于帖子清理排序，不是独立赞踩系统；真正赞踩仍写入玩家口碑系统。

## 存储方式

帖子系统使用独立数据库表，定义在：

```text
mdtserver/config/scripts/wayzer/lib/MdtStorage.kt
```

当前表：

- `MdtForumSections`：帖子分区定义。
- `MdtForumPosts`：帖子主体。
- `MdtForumComments`：帖子评论。
- `MdtForumAuthorStats`：玩家历史累计发帖数，用于后续成就/排行扩展。

配置/状态：

- `forum.cleanup.lastDate`：最近一次每日清理日期，存放在 `MdtSettings`。
- `forum.stats.totalPosts`：历史累计总发帖数，存放在 `MdtSettings`。
- `forum.stats.initialized.v1`：发帖统计初始化标记，存放在 `MdtSettings`。
- `forum.lockedPostIds`：自动清理锁定帖子 ID 列表，存放在 `MdtSettings`。
- `forum.protectedPostIds`：4级/admin 保护锁帖子 ID 列表，存放在 `MdtSettings`。
- `forum.postChangeHistory`：最近 10 条玩家修改/删除帖子记录，存放在 `MdtSettings`。
- `forum.deleted.<帖子ID>.by/reason/at`：回收站删除者、原因和时间，存放在 `MdtSettings`。

## 成就埋点

- 发帖成功后会触发 `ForumPostCreatedEvent`。
- 当前已接入成就：
  - `first_forum_post` / **首次发贴**：首次发帖，奖励 `5` MDC。
- 后续可基于 `MdtForumAuthorStats.total_posts` 添加累计发帖类成就或排行。

## 编辑超时

- 帖子菜单与文本输入框的服务端等待时间已调整为约 30 分钟，便于玩家编辑较长正文。
- 如果客户端自身因长时间无操作关闭弹窗，服务端脚本无法强制覆盖。

## 维护建议

- 暂不做楼中楼，避免菜单和数据库复杂度过高。
- 暂不做独立帖子赞数，避免绕开玩家赞踩限制。
- 若未来帖子量明显变大，可考虑把数据库从默认 H2 切换到 PostgreSQL。
