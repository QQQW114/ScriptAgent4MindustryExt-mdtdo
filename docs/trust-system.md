# 信任/口碑系统指令与规则

本文档记录本轮新增与已添加的信任等级、赞踩、认可、MDC（MDT DO Credit）相关内容。后续新增指令时继续维护此文件。

概念区分：

- **信任等级**：表示服务器对玩家的信任程度，用于帖子/Wiki、投票否决、管理权限、处罚与安全边界。
- **资历等级**：表示玩家在服务器玩法方向上的成长资历，主要服务于技能系统与技能商店，不授予帖子或管理权限。

## 玩家指令

- `/account`、`/账号`：打开账号注册/登录菜单。
- `/register`、`/注册`：注册 QQ 账号并自动登录；注册前游客 UUID 已落库的当前/累计 MDC 会合并到新账号主体，来源游客 MDC 清零/删除以避免重复使用。
- `/login`、`/登录`：登录已有 QQ 账号。
- `/changepassword`、`/passwd`、`/改密`、`/修改密码`：修改当前账号密码。
- `/deleteownaccount`、`/注销账号`、`/账号注销`：注销当前登录账号，需要当前密码与确认菜单。
- `/points`、`/积分`、`/mdc`、`/credit`：查看自己的当前MDC与累计MDC。
- MDC发生获得/失去时，在线玩家会收到仅自己可见的统一短提示，例如 `MDC-10（技能消耗）`。
- `/seniority`、`/资历`、`/playtime`、`/在线时长`：查看自己的资历等级、累计在线时长、当前/累计MDC与下一资历等级条件。
- `/pay <玩家id/3位id> <数量> [留言]`：向其他玩家转账MDC；别名 `/transfer`、`/转账`、`/付款`、`/mdcpay`。
- `/redpacket <总MDC> <份数> [留言]`：发MDC红包；别名 `/hongbao`、`/hb`、`/红包`、`/发红包`。
- `/grab <红包ID>`：抢MDC红包；不填ID时显示当前可抢红包；别名 `/抢红包`、`/开红包`、`/抢`、`/grabredpacket`。
- `/redpackets [all]`：查看红包列表；别名 `/红包列表`、`/hblist`。
- `/recognize <玩家id/3位id>`、`/认可 <玩家id/3位id>`：认可一名玩家。
- `/playerinfo [玩家id/3位id/#游戏id]`、`/pinfo`、`/玩家信息`：打开玩家信息/交互面板；不填目标时弹出在线玩家列表。
- `/title`、`/称号`、`/titles`：打开称号面板，佩戴/取消佩戴已拥有的正式称号。
- `/achievements`、`/achievement`、`/成就`、`/成就系统`：打开成就系统。
- `/shop`、`/商店`、`/shops`：打开商店列表。
- `/titleshop`、`/称号商店`：打开称号商店。
- `/skillshop`、`/技能商店`：打开技能商店。
- `/skill`、`/skills`、`/技能`：打开技能分类菜单。
- `/randomform`、`/随机形态`、`/变换形态`：随机变换形态；已有特殊形态时再次使用会恢复原状态。

## 管理指令

- `/setlevel <uuid/3位id> <0|1|2|3|3+|4>`：设置玩家信任等级。
- `/locklevel <uuid/3位id> [on|off|toggle]`：锁定/解除玩家信任等级自动调整。
  - 别名：`/levellock`、`/等级锁`、`/锁等级`
  - 锁定后该玩家等级不会被晋升系统自动晋升或降级；解除后会在下一次晋升检测时恢复自动控制。
- `/setadmin <在线玩家id/3位id/#id/名字> [on|off|toggle]`：设置/取消 Mindustry 原生管理员。
  - 别名：`/adminplayer`、`/op`、`/设管理`
- `/reputation <玩家id/3位id> [赞|踩] [set|add] [数量]`：查看或修改玩家被赞/被踩数量。
  - 别名：`/rep`、`/口碑`、`/赞踩`
- `/trustpoint <玩家id/3位id> [set|add|spend|totalSet|totalAdd] [数量]`：查看或修改玩家MDC。
  - 别名：`/tpoint`、`/改积分`、`/改mdc`（`/改积分` 保留兼容旧用法）
- `/servertestmode [status|on confirm|off confirm|menu]`：`[危险]服务器测试模式`，特殊测试服临时账号/MDC/资历覆盖。
  - 权限：`wayzer.admin.serverTestMode`，默认 `@admin`。
  - 启用时仅已登录账号玩家进入临时测试主体；正式账号本身为信任4级或原生 admin 的玩家仍显示为4级，其余已登录测试玩家临时视为信任 `1`、资历 `3`。未登录游客仍为0级且不获得测试MDC。
- `/trustcheck [玩家id/3位id]`：手动检查玩家信任等级条件，并查看相关统计。
- `/setseniority <uuid/3位id> <0|1|2|3|4>`：设置并锁定玩家资历等级；别名 `/设置资历`、`/调整资历`。
- `/lockseniority <uuid/3位id> [on|off|toggle]`：锁定/解除玩家资历等级自动调整；解除锁定后会重新按在线时长与累计MDC自动检测。别名 `/senioritylock`、`/资历锁`、`/锁资历`。
- `/setplaytime <uuid/3位id> <小时>`：按小时设置玩家累计在线时长。
- `/addplaytime <uuid/3位id> <小时>`：按小时增加玩家累计在线时长。
- `/playermute <玩家id/3位id/#游戏id> [理由]`：禁言在线玩家。
  - 别名：`/pmute`、`/mute`、`/禁言`
- `/playerunmute <玩家id/3位id/#游戏id>`：解除在线玩家禁言。
  - 别名：`/punmute`、`/unmute`、`/解除禁言`
- `/buildban <玩家id/3位id/#游戏id> [理由]`：禁止在线玩家建造/拆除；玩家信息面板使用时可输入临时时长，留空为永久。
  - 别名：`/nobuild`、`/禁建`、`/禁止建造`
- `/buildunban <玩家id/3位id/#游戏id>`：解除在线玩家建造/拆除限制。
  - 别名：`/allowbuild`、`/解除禁建`、`/允许建造`
- `/playertitle <玩家id/3位id> [list|grant|revoke|equip|clear] [code] [显示名]`：查看、授予、撤销或调整玩家称号。
  - 别名：`/titleadmin`、`/givetitle`、`/称号管理`
- `/titledef <code> <显示名>`：定义或更新称号代码对应的显示名。
  - 别名：`/deftitle`、`/定义称号`
- `/achadmin <玩家id/3位id> [list|check|grant|revoke] [成就code]`：查看、检测、授予、撤销玩家成就。
  - 别名：`/achievementadmin`、`/成就管理`
- `/titleshopadmin list|set|del|seed ...`：管理称号商店商品。
  - 别名：`/tshopadmin`、`/称号商店管理`
  - 常用格式：`/titleshopadmin set <商品id> <称号内容|custom:长度> <售价> <等级要求> [认可要求]`
- `/setpassword <QQ账号> [新密码]`：重置玩家账号密码；玩家管理员使用时会弹出确认菜单，控制台用法为 `setpassword <QQ账号> <新密码> confirm`。
  - 别名：`/resetpassword`、`/账号改密`、`/重置密码`
- `/deleteaccount <QQ账号/玩家3位ID/UUID/account:id> [confirm]`：删除玩家账号；玩家管理员使用时会弹出确认菜单，控制台用法为 `deleteaccount <QQ账号/玩家UUID/account:id> confirm`。
  - 别名：`/delaccount`、`/账号删除`、`/删除账号`
- `/accountqq <玩家3位ID/UUID/account:id/QQ账号>`：查询玩家对应 QQ 账号。
  - 别名：`/qqof`、`/查qq`、`/查询qq`、`/账号查询`、`/queryqq`
- `/guestob [on|off|status]`：查看/手动设置今日未登录玩家强制观战。
  - 别名：`/游客观战控制`、`/guestforceob`
- `/veto [原因]`：3+级与4级玩家一票否决当前正在进行的投票。
  - 别名：`/否决`、`/一票否决`、`/一票否决权`

投票限制：

- `/vote` 默认对玩家开放，信任 `0`级/未登录玩家也可发起和参与投票。
- 投票失败后会进入发起冷却；未登录/游客使用更长失败冷却，同 IP 共享发起冷却，参与计票同 IP 只按最后一票生效。
- 信任 `4`级/管理员拥有 `wayzer.admin.voteUnlimited`，发起投票时不受失败冷却和同 IP 发起冷却限制；但投票系统仍保持同一时间只能有一个正在进行的投票。

## 赞踩额度与有效被踩

每日赞/踩额度按信任等级计算，`3+` 在额度上按 `3` 级处理：

| 信任等级 | 点赞：每目标/每日总量 | 点踩：每目标/每日总量 |
| --- | --- | --- |
| `1` | `3 / 10` | `2 / 5` |
| `2` | `5 / 25` | `3 / 8` |
| `3` / `3+` | `8 / 50` | `5 / 12` |
| `4` | 不限 | 不限 |

实现注意：

- 0级/未登录玩家不能赞踩。
- 同 IP 账号之间不能点赞，避免刷赞/MDC；同 IP 认可也会被拦截。
- 每日额度检查与写入在 `MdtStorage.recordReputationVoteChecked(...)` 的单个事务内完成，避免先查后写导致上限异常。
- `MdtReputationDaily` 保留最近 14 天；若历史上存在重复每日计数行，当前写入会按重复行求和，成功后合并重复数据，避免出现“只能点 1-2 次/上限随机刷新”的现象。

有效被踩用于信任晋升与玩家信息面板，当前公式为：

```text
有效被踩 = max(0, 总被踩 + 最近7天被踩*2 - 被认可*2 - 总被赞/20 - 最近7天被赞/3)
```

该公式会更重视最近几天的被踩情况，并大幅降低“历史总被赞”对有效被踩的抵消效果，以便更好评估玩家近期“不被欢迎”的程度。

## 资历等级

资历等级用于技能分类、技能商店技能购买/使用与管理员技能入口；信任等级仍控制帖子、Wiki/帖子管理、投票否决、管理权限、处罚等信任边界。

- 资历 `0`：仅使用通用技能。
- 资历 `1`：累计在线至少 `1` 小时，且累计MDC至少 `100`。
- 资历 `2`：累计在线至少 `16` 小时，且累计MDC至少 `600`，可使用 2级技能。
- 资历 `3`：累计在线至少 `64` 小时，且累计MDC至少 `2000`，可使用 3级技能。
- 资历 `4`：可使用管理员技能；不会自然晋升，信任等级 `4`/已登录admin 默认视为资历 `4`，也可由管理员手动设置并锁定。

在线时长只统计已登录账号主体，按小时显示；转账与红包不增加累计MDC，因此不能绕过资历累计MDC条件。

测试模式例外：`[危险]服务器测试模式` 启用后，仅已登录账号玩家会切到 `test:<account:id>` 临时主体。正式账号本身为信任4级或原生 admin 的玩家仍视为4级，其余已登录测试玩家临时视为信任 `1`、资历 `3`；未登录游客仍为0级。测试 UID 不会触发正式信任/资历自动晋升，MDC 写入临时测试数据，不写正式 MDC 表。

## 当前MDC来源与流转

- 注册迁移：未登录游客注册新账号成功后，会把该 UUID 主体已落库的当前MDC与累计MDC合并到 `account:<id>`；只迁移 MDC，不迁移游客 UUID 上的手动信任等级/等级锁。
- 一局游戏贡献结算：要求本局时长至少 `15` 分钟，且个人在线超过 `60` 秒、活跃至少 `60` 秒；按贡献分发 MDC，活跃约每 `10` 分钟 `+2` MDC，PVP胜利最多约额外 `+2` MDC，单局最多 `48` MDC。
  - `[危险]服务器测试模式` 启用时，本局贡献结算奖励额外 `×10`，且写入临时数据文件；关闭测试模式后清理。
- 结算后发送 `gg`：`+3` MDC，要求本局时长至少 `5` 分钟，每局每名玩家只奖励一次。
- 收到一次赞：`+5` MDC；同IP账号之间不能互相点赞，因此不会触发该奖励。
- 收到一次认可：`+25` MDC；同IP账号之间不能互相认可，因此不会触发该奖励。
- 随机变换形态：每天每名玩家最多获得一次MDC。
  - 猫娘：`+15` MDC。
  - chatgpt：`+10` MDC。
  - 嘉豪：`+5` MDC。
  - 形态聊天改写会保留玩家原话末尾单个 `(` 或 `（` 到最终文本末尾，例如 `大家好（` 会变为 `大家好喵（`。
- 成就MDC：详见 `docs/achievement-system.md`。

玩家间流转：

- 转账与红包只改变“当前MDC”，不会增加接收者的“累计MDC”。
- 转账：发送者当前MDC减少，接收者当前MDC增加。
- 红包：发包时先扣除发包者当前MDC；每份至少 `1 MDC`；单个红包最多 `50` 份。
- 红包 `10` 分钟后过期，未抢完的剩余MDC会退回发包者当前MDC。
- 发包者不能抢自己的红包；同一玩家对同一个红包只能抢一次；未登录游客不能抢红包。
- 红包抢完时会广播领奖者列表、手气王和手气榜。

说明：管理指令手动修改被赞/被踩数量只用于修正统计与触发等级检查，不自动补发“收到赞”的MDC；如需补MDC请使用 `/trustpoint`。同IP判断优先使用在线连接IP，离线目标使用账号主体最后登录IP。

## 持久化说明

信任等级、资历等级/在线时长、等级锁、MDC、赞/踩、每日赞踩额度、认可、称号、成就、随机形态、禁言和红包均已切换到数据库持久化；临时禁言/临时禁建只保存在内存，永久禁建保存为脚本 Savable 状态。

- 默认数据库：`mdtserver/config/scripts/data/h2DB.db`。
- 存储层：`mdtserver/config/scripts/wayzer/lib/MdtStorage.kt`。
- 表注册脚本：`mdtserver/config/scripts/wayzer/mdtDatabase.kts`。
- 表结构见 `docs/database-system.md`；账号注册/登录与未登录观战投票见 `docs/account-system.md`。

## 权限说明

- 信任等级 `4` 的在线玩家会被加入 `@admin` 权限组，可以使用注册给管理员组的管理指令。
- 已登录的在线 Mindustry admin 视为信任等级 `4`；未登录会话仍按游客处理。
- 称号管理权限 `wayzer.admin.playerTitle` 默认注册给 `@admin`。
- 成就管理权限 `wayzer.admin.achievement` 默认注册给 `@admin`。
- 称号商店管理权限 `wayzer.admin.titleShop` 默认注册给 `@admin`。
- 账号管理权限 `wayzer.admin.account` 默认注册给 `@admin`。
- 信任等级管理权限 `wayzer.admin.trustLevel` 默认由 `@admin` 的 `wayzer.*` 通配权限覆盖；`/setlevel`、`/locklevel`、`/setadmin` 仍会在指令体内二次确认当前会话已达到信任4级/admin。
- `/veto` 不走普通权限组，而是直接检查玩家信任等级是否达到 `3+` 或 `4`。
- MDC管理权限 `wayzer.admin.trustPoint` 默认注册给 `@admin`。
- 资历管理权限 `wayzer.admin.seniority` 默认注册给 `@admin`。

## 信任等级简述

- `0`：默认未登录/游客；可登录/注册、查看基础菜单、发起/参与投票（失败后游客冷却更长、同IP共享发起冷却），但不能私聊、发帖/评论、赞踩、认可，也不能继承历史 UUID/admin 权限。
- `1`：已注册并登录账号的玩家；可发帖/评论、使用私聊、点赞/点踩（1级额度）、查看/发布 1级分区内容，并可在玩家面板对 `0` 级目标发起投票踢出/投票强制观战。
- `2`：有一定MDC与被认可度的玩家；可认可他人，点赞额度提高，可查看 2级分区内容，并可在玩家面板直接强制观战 `0` 级目标。
- `3`：更高活跃度与认可度的玩家；可查看 3级分区内容，点赞/点踩额度按 3级档计算，可购买/使用需要信任3级的称号商店内容。
- `3+`：独立等级，位于 `3` 与 `4` 之间；可查看 3+分区，可使用 `/veto` 一票否决，可管理 Wiki/帖子中的普通内容，可直接开关性能优化/服务器压力相关措施、管理封禁地图/今日PVP，并可对低于 3+ 的目标直接强制观战、禁言/解禁。
- `4`：管理员层级，对应 `@admin`，不通过自然晋升获得；拥有注册给管理员组的管理指令和通配权限，可设置信任/资历/MDC/称号/成就/账号、封禁账号/IP、查看最近玩家/IP封禁列表、使用管理员频道/神权菜单/世界处理器/CP等管理入口。已登录的原生 admin 也视为 `4+admin`。

信任等级不再作为普通技能/2级技能/3级技能的限制依据；技能资格改由“资历等级”判断。
